## ❓ Common Demo Questions & Answers

**Q: "Why React and not Vue or Svelte?"**
A: "React is widely used, has great ecosystem support, and we're familiar with it. For this project's scope, the component model and Context API provide the right balance of simplicity and power."

**Q: "Why Express and not Next.js?"**
A: "We needed a separate backend for security (to hide the API key). Express gives us fine control over the API design and authentication without extra Next.js complexity."

**Q: "How does the AI know about the user?"**
A: "The chat sends context including the user's fitness profile, their plan, and completed workouts. This way, the AI can give personalized advice."

**Q: "What happens if the API fails?"**
A: "We have error handling everywhere. For plan generation, we show a fallback plan. For chat, we show a friendly error. The user experience degrades gracefully."

**Q: "How is the API key protected?"**
A: "It's stored in a .env file on the server, never sent to the browser. All API requests go through our backend proxy, which adds the key server-side."

**Q: "How do you manage state?"**
A: "We use React Context API to store user profile, fitness plans, completed tasks, and chat history. This is simpler than Redux for our use case."
